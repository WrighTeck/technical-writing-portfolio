# QA Debugging Tools for Web, Mobile & OTT Testing

This section documents practical usage of free debugging tools commonly used in real-world QA workflows.

These tools support:

- Network traffic inspection
- API validation
- Authentication debugging
- Performance troubleshooting
- Front-end console investigation
- OTT playback and ad beacon validation

---

# Tools Covered

- Charles Proxy – Network traffic inspection & API validation  
- Chrome DevTools – Front-end debugging & performance analysis  
- Postman – API testing & authentication validation  

---

# 1. Charles Proxy

Charles Proxy is used to inspect network traffic between client applications and backend services.

It is especially useful for:

- API request/response validation
- Header inspection
- Token validation
- Environment configuration debugging
- OTT ad beacon tracking
